<p>This tutorial works best if you are using the Bash shell. Linux and macOS come standard with Bash, which you can access in the Terminal application. Windows PowerShell uses the .NET framework, which is not Bash. For Windows users, you have several options:</p>
<ol>
<li>Best option: Install <a href="https://gitforwindows.org" rel="nofollow">Git Bash</a> and use the Bash shell that comes with it. This is fast.</li>
<li>Next best option: Install a Linux virtual machine like <a href="https://www.virtualbox.org" rel="nofollow">VirtualBox</a> and use the Terminal that comes with it. This takes more time but gives you access to a virtual Linux machine on your PC.</li>
<li>Last resort: Use Windows PowerShell. This has some functionality, as you saw in Appendix A of Shaw. <h4><a id="user-content-introduction" class="anchor" aria-hidden="true" href="#introduction"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Introduction</h4>
<p>Nano is a text editor suited to working in UNIX. It is not as powerful as PC window-based editors, as it does not rely on the mouse, but still has many useful features.
Most nano commands are invoked by holding down the <code>Ctrl</code> key (that is, the control key), and pressing one of the other keys. In this text, the control key is referred to using <code>^</code>. For example, <code>^X</code> means "hold down the control key and press the x key". Most of the important commands are listed at the bottom of your screen.</p>
<ul>
<li><code>^G</code> nano help</li>
</ul>
<h4><a id="user-content-starting-nano" class="anchor" aria-hidden="true" href="#starting-nano"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Starting nano</h4>
<p>To edit a file called filename, type nano filename.
In nano, you can insert another file:</p>
<ul>
<li><code>^R</code>	read an existing file into nano (inserted at the current cursor position)</li>
<li><code>^T</code>	opens a browser that allows you to select a file name from a list of files and directories</li>
</ul>
<h4><a id="user-content-navigation" class="anchor" aria-hidden="true" href="#navigation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Navigation</h4>
<p>The usual mouse-based point-and-click method is not supported by nano. Use the arrow keys to move around the page in nano.</p>
<p>Other navigation commands:</p>
<ul>
<li><code>^A</code>	move to beginning of line</li>
<li><code>^E</code>	move to end of line</li>
<li><code>^Y</code>	move down a page</li>
<li><code>^V</code>	move up a page</li>
<li><code>^_</code>	move to a specific line (<code>^_^V</code> moves to the top of the file, <code>^_^Y</code> to the bottom)</li>
<li><code>^C</code>	find out what line the cursor is currently on</li>
<li><code>^W</code>	search for some text.</li>
</ul>
<p>When searching, you will be prompted for the text to search for. It searches from the current cursor position, wrapping back up to the top if necessary.</p>
<h4><a id="user-content-editing" class="anchor" aria-hidden="true" href="#editing"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Editing</h4>
<p>Insert new text at the current cursor position just by typing the text in.</p>
<p>Delete commands:</p>
<ul>
<li><code>^D</code>	delete character currently under the cursor</li>
<li><code>Backspace</code>	delete character currently in front of the cursor</li>
<li><code>^K</code>	delete entire line</li>
<li><code>^\</code>	search for (and replace) a string of characters</li>
</ul>
<p>Cut and paste:</p>
<ul>
<li><code>^K</code> does not delete lines permanently; the most recent set of deletions are stored in a buffer. These lines may be re-inserted at the current cursor location using <code>^U</code>. This may be used to simulate cut and paste:
<ul>
<li>Repeatedly use <code>^K</code> until all of the text you want to move has been deleted.</li>
<li>Move to the line that you want to insert the text at, and use <code>^U</code>.</li>
</ul>
</li>
<li>Note that pressing <code>^U</code> more than once will cause multiple copies to be inserted. This is particularly useful if you want to copy text:
<ul>
<li>Repeatedly use <code>^K</code> until all of the text you want to copy has been deleted.</li>
<li>Press <code>^U</code> immediately to put a copy back in its original location.</li>
<li>Move to the line that you want to copy the text to, and use <code>^U</code>.</li>
</ul>
</li>
</ul>

<ul>
<li><code>^O</code>	save contents without exiting (you will be prompted for a file to save to)</li>
<li><code>^X</code>	exit nano (you will be prompted to save your file if you haven't)</li>
<li><code>^T</code>	when saving a file, opens a browser that allows you to select a file name from a list of files and directories</li>
</ul>
